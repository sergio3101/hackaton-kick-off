## Найденные баги (для `doc/improvements.md`)

### P0 — нарушение жёсткого бана LLM
- **C-9 [!]** `POST /api/requirements/{id}/regenerate` с `selected_topics=[]` НЕ блокируется (200 OK). По факту вызвал `generate_questions_for_pair` × 6 тем × 3 уровня = ~18 LLM-вызовов и **перезаписал весь `question_bank`** для requirement #1. Старые prompt-тексты потеряны, новые id (115..144 и др.). Темы те же по имени — сессии не сломались.
  - **Ожидание:** 400 / 422 на пустой список тем (или явное «`["all"]`» как opt-in).
  - **Фикс:** валидация `len(selected_topics) >= 1` в Pydantic-схеме `RequirementsRegenerateIn`.

### P1 — мягкие валидации coding sandbox
- **F-5 [!]** `POST /coding/run` принимает любой `lang`, игнорирует параметр и запускает с `coding_language` задачи. Ожидание: 400 на неподдерживаемый.
- **F-6 [!]** Пустой `code` → 200 + успешное выполнение пустого скрипта. Ожидание: 400.
- **F-4 [~]** Truncate threshold не достигнут на 100K строк (стоит уточнить лимит — возможно слишком высокий).

### P1 — мягкие валидации REST
