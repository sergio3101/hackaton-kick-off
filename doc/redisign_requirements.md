# Требования к редизайну платформы kick-off собеседований с AI-агентами

## 1. Цели редизайна

- Снизить порог входа для нового пользователя: первый kick-off — не позднее чем через 2 минуты после регистрации.
- Повысить воспринимаемое качество голосового взаимодействия (естественность, отзывчивость, отсутствие "роботизированности").
- Сделать аналитику и историю основным рабочим инструментом, а не "архивом", куда заходят редко.
- Обеспечить масштабирование: от одиночного пользователя до команды из 50+ человек без перестройки UX.
- Унифицировать визуальный язык (сейчас, предположительно, разные части продукта выглядят как разные продукты).

## 2. Целевые пользователи и сценарии

**Основные роли:**
- Заказчик/инициатор — проводит kick-off с клиентом или внутренней командой, собирает требования.
- Аналитик/PM — изучает историю, ищет паттерны, формирует отчёты.
- Руководитель — смотрит дашборд, контролирует активность команды, оценивает качество.
- Администратор — управляет доступами, шаблонами, AI-агентами.

**Ключевые сценарии:**
- "Запустить kick-off за 60 секунд" — выбрать шаблон, агента, нажать старт.
- "Найти, что обсуждалось с клиентом X в марте" — поиск по истории.
- "Показать руководителю, как идут дела по проекту" — дашборд + экспорт.
- "Сравнить два kick-off между собой" — аналитика.

## 3. Главный функционал — голосовые kick-off собеседования

### 3.1 Инициация и подготовка
- Библиотека шаблонов kick-off по типам (продуктовый, проектный, дискавери, customer-интервью и т.д.).
- Конструктор сценария: цели встречи, обязательные вопросы, опциональные ветки, ограничения по времени.
- Выбор AI-агента: персона, тон, язык, темп речи, голос (мин. 4 голоса на каждый поддерживаемый язык).
- Превью-режим: 30 секунд диалога с агентом до запуска "боевой" сессии.
- Приглашение участников по ссылке без регистрации (с ограниченными правами).

### 3.2 Голосовое взаимодействие
- Задержка ответа агента — не более 800 мс от конца реплики пользователя.
- Корректная обработка перебиваний (barge-in): агент останавливается, не дожидаясь конца своей фразы.
- Распознавание эмоциональной окраски и адаптация темпа/тона ответа.
- Поддержка минимум 5 языков с автоопределением языка реплики.
- Работа в шумной среде (фильтрация фонового шума, эхо, голосов рядом).
- Индикация состояния агента: слушает / думает / говорит — визуально и тактильно (на мобильных).
- Возможность письменного ввода в любой момент без выхода из голосового режима.

### 3.3 Управление в ходе сессии
- Пауза/возобновление без потери контекста.
- "Подсказки" агенту от ведущего в реальном времени (скрытый канал — "спроси про бюджет").
- Заметки и теги на лету, привязанные к таймкоду.
- Live-транскрипт с возможностью редактирования после.
- Аварийный выход с сохранением всех данных.

### 3.4 Завершение
- Автоматическое резюме: цели, договорённости, открытые вопросы, action items с ответственными.
- Подтверждение/правки резюме перед сохранением.
- Автоотправка участникам выбранным каналом (email, ссылка, мессенджер).

## 4. История собеседований

- Список с превью: дата, длительность, участники, агент, тип, ключевые темы, статус.
- Множественная фильтрация: по дате, типу, агенту, участникам, тегам, длительности, статусу.
- Полнотекстовый поиск по транскриптам и резюме с подсветкой совпадений.
- Семантический поиск ("найди, где обсуждали сроки запуска") — не только по ключевым словам.
- Карточка собеседования: транскрипт с таймкодами, аудио с возможностью кликнуть по фразе и услышать её, заметки, теги, резюме, аналитика.
- Воспроизведение: ускорение (0.5x–2x), пропуск пауз, поиск по транскрипту в плеере.
- Экспорт: PDF (резюме), DOCX (транскрипт), MP3 (аудио), JSON (структурированные данные).
- Версионность: если резюме редактировалось, хранится история изменений.
- Связи между собеседованиями: "продолжение", "связанные", "по тому же проекту".
- Архивирование и удаление с soft-delete (30 дней).

## 5. Аналитика

### 5.1 По одному собеседованию
- Распределение времени между говорящими.
- Структура диалога по темам (визуальная карта).
- Эмоциональный профиль во времени.
- Скорость речи, паузы, перебивания.
- Покрытие планируемых вопросов (что спросили, что пропустили).
- Извлечённые сущности: люди, компании, даты, цифры, продукты.

### 5.2 Агрегированная
- Динамика по периодам (день/неделя/месяц/квартал).
- Сравнение проектов, команд, агентов между собой.
- Топ-темы за период, рост/падение упоминаний.
- Эффективность шаблонов: какой шаблон даёт самые полные kick-off.
- Качество агентов: рейтинг, отказы пользователей продолжать, среднее время сессии.

### 5.3 Инсайты
- Автоматические наблюдения ("в 80% kick-off с клиентом X поднимается тема бюджета").
- Алерты при отклонениях ("длительность kick-off в этом месяце выросла на 40%").
- Рекомендации по улучшению шаблонов на основе данных.

## 6. Дашборд

- Кастомизируемая сетка виджетов (drag-and-drop), несколько сохранённых раскладок.
- Базовый набор виджетов: активность за период, последние сессии, ближайшие запланированные, KPI команды, топ-теги, статус action items.
- Быстрые действия: "начать kick-off", "открыть последнее", "пригласить участника".
- Уведомления: новые комментарии, упоминания, действия команды.
- Режим "дайджест" — еженедельная сводка на email/в мессенджер.
- Разные представления для разных ролей (руководитель видит команду, исполнитель — себя).

## 7. UX/UI требования

- Единый визуальный язык: токены цветов, типографики, отступов, анимации.
- Тёмная и светлая темы, переключение без перезагрузки.
- Адаптивность: десктоп, планшет, мобильный (мобильный — обязательно для проведения kick-off, а не только просмотра).
- Доступность по WCAG 2.1 AA: контраст, навигация с клавиатуры, скринридеры, субтитры в реальном времени.
- Микровзаимодействия: визуализация голоса (waveform), плавные переходы, "дыхание" агента.
- Состояния пустоты, ошибок, загрузки — продуманы для каждого экрана.
- Онбординг: интерактивный тур + примеры готовых kick-off.
- Локализация интерфейса минимум на те же языки, что поддерживает голосовой агент.

## 8. Нефункциональные требования

- Производительность: первый интерактив ≤ 2 с, переход между экранами ≤ 300 мс.
- Доступность системы: 99.5% uptime, отдельный SLA на голосовой канал — 99.9%.
- Безопасность: SSO (Google, Microsoft, SAML), 2FA, шифрование данных в покое и в передаче.
- Соответствие GDPR: согласие на запись, право на удаление, экспорт всех данных пользователя.
- Хранение записей: настраиваемый срок (30 дней / 1 год / бессрочно), автоудаление.
- Аудит-лог: кто, когда, что смотрел/менял.
- Резервное копирование: ежедневное, восстановление до точки за последние 14 дней.

## 9. Технические требования

- API для интеграций (REST + Webhooks): Slack, Teams, Notion, Jira, CRM.
- SDK для встраивания виджета kick-off на сторонние сайты.
- Кросс-браузерность: последние 2 версии Chrome, Safari, Firefox, Edge.
- Нативные приложения iOS/Android — как минимум для проведения сессий и просмотра истории.
- Поддержка корпоративных ограничений: VPN, прокси, белые списки доменов.
- Возможность on-premise развёртывания для enterprise-клиентов (отдельный трек).

## 10. Метрики успеха редизайна

- Time-to-first-kickoff для нового пользователя: цель ≤ 2 минуты.
- Активация: % новых пользователей, проведших ≥ 1 kick-off в первую неделю.
- Возвращаемость: % пользователей, возвращающихся к истории/аналитике в течение месяца.
- NPS / CSAT — отдельно по голосовому опыту и по продукту в целом.
- Доля сессий, завершённых до конца (не прерванных).
- Среднее количество просмотров одного завершённого kick-off (показатель ценности истории/аналитики).

## 11. Что важно решить до старта работы

- Скоуп MVP редизайна: что входит в первую версию, что переносится.
- Миграция данных и пользователей со старой версии — стратегия и сроки.
- Подход к версионированию: параллельный запуск старой и новой или жёсткое переключение.
- Бренд: остаётся текущая визуальная идентичность или ребрендинг внутри редизайна.
- Бюджет голосовой инфраструктуры (LLM + TTS/STT) и его влияние на UX-решения (например, ограничения на длину сессии).
