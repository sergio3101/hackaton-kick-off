## Готово / решено

- [x] **P1** **STT падает при коротком/пустом аудио** — на фронте `useVoiceSession` теперь проверяет `MIN_RECORDING_MS=600` / `MIN_BLOB_BYTES=1500` и не отправляет пустышки; на бэке `llm/voice.py` ввёл `AudioTooShortError`, ловит его и `BadRequestError(audio_too_short)` от Whisper; WS возвращает `{"type":"error","code":"audio_too_short","recoverable":true}`, диалог не падает.
- [x] **P1** **Сырые тексты ошибок в голосовой панели** — `VoicePanel` показывает короткое сообщение в отдельной плашке с кнопкой «Закрыть»; ошибки от сервера несут `code/recoverable`, recoverable-ошибки автоматически возвращают phase в `listening`.
- [x] **P1** **Очевиднее управление микрофоном** — заменил hold-to-talk на круглую toggle-кнопку микрофона с пульсацией при записи и понятной подписью «Нажмите, чтобы начать запись» / «Идёт запись — нажмите ещё раз, чтобы отправить»; добавил цветной индикатор фазы.
- [x] **P1** **Просмотр и редактирование проектов** — добавил страницу `/requirements/:id` (`RequirementsDetail.tsx`) с summary/темами/банком вопросов и счётчиками 5/5 на пару. Эндпоинты на бэке: `GET/PATCH/DELETE /api/requirements/:id` и `POST /api/requirements/:id/regenerate`. Под-пункт «не начинать сразу» — закрыт ниже.
- [x] **P1** **Не начинать вопросы автоматически** — на странице `/sessions/:id/interview` показывается заглушка с кнопкой «Начать интервью»; до клика WS не подключается, кодинг-задача доступна сразу.
- [x] **Голос — заменить на более человечный** — в `.env` `OPENAI_TTS_VOICE` переключён `alloy → nova`.
- [x] **P1** **Слишком мало вопросов на пару тема×уровень** — ужесточил `EXTRACT_SYSTEM` (явное «РОВНО 5 вопросов на каждую пару, отклонения недопустимы»), добавил `TOPIC_QUESTIONS_SYSTEM`/схему и функцию `generate_questions_for_pair` в `llm/extract.py`. Новая `ensure_full_bank` догенерирует недостающие вопросы по любой неполной паре. Вызывается и при загрузке, и при `regenerate`.
- [x] **Лайв-кодинг по теме** — на сессию генерируются ровно 3 кодинг-задачи, каждая привязана к теме сессии (если тем < 3 — повторение, если > 3 — сэмплинг без повторов). `CODING_TASKS_SYSTEM`/схема + `generate_coding_tasks` в `llm/generate.py`; в `routers/sessions.py` добавлены `_pick_coding_topics`, `CODING_TASKS_PER_SESSION=3`. Эндпоинт ревью переехал на `POST /api/sessions/{sid}/coding/review/{item_id}` и берёт prompt из конкретного `SessionQuestion`. На фронте `CodingPanel` показывает вкладки `[ #1 Topic ] [ #2 Topic ] [ #3 Topic ]` с отдельным редактором, кодом, verdict-индикатором и результатом для каждой задачи.
- [x] **Базовое серверное логирование** — `app/logging_setup.py` (`setup_logging()`), `LOG_LEVEL` в `config.py`/.env.example, инициализация в `main.py`. Логи INFO в роутерах `requirements`/`sessions` (загрузка, regenerate, delete, create_session с topics/voice/coding, start, submit_coding, finish) и в LLM-модулях (`extract`, `generate`, `evaluate`, `voice`) — без полного payload, только размеры/числа/verdict. Уровень доступа uvicorn приглушён до WARNING.
- [x] **Кнопка «Отправить ответ» + многосегментная запись** — в `useVoiceSession` запись больше не отправляется автоматически на stop; toggle микро = старт/пауза, накопленные чанки копятся в `chunksRef`, новый метод `submitAnswer()` склеивает их и шлёт одним blob в WS. Появился `discardSegments()` и счётчик `segments` в state. В `VoicePanel` рядом с микро добавлены кнопки «Отправить ответ» (primary) и «Очистить» (secondary), под микрофоном — динамическая подпись и индикатор «Записано сегментов: N». Минимальные пороги длительности/размера применяются к сумме всех сегментов.
